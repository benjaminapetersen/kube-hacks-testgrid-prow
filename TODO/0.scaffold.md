# Scaffolding!

Per the README:

    Hacking around with a fresh idea for a single UI for Test Grid and Prow.
    Given Headlamp is now under SIG-UI and vendor neutral I'll model the
    frontend deps of this project to follow their lead.

    And for fun, asked Copilot to quick cough up some matching logos for
    this project, based on the existing Prow logo, but modernized a bit.
    These are not official in any way.

So we will need a few things.

1. A Backend that may act only as a proxy for now, to get the bits we need
  from Prow and Test Grid.

2. A Frontend that is built with similar or the same tools as Headlamp and
  has a similar layout and structure when appropriate, such that it is clear
  that the projects are all managed by Kubernetes

## The Plan: Frontend

We want a single UI "pane of glass", but with clear distinction between
Prow and Test Grid, utilizing the 2 logos, but ensuring the navigation is
crystal clear and not confusing.

We want to model after Headlamp's look and feel.  Headlamp is a UI for
Kubernetes itself, so it has different concerns, but nevertheless it would
be good to be similar.  We should change logos and change color scheme basics
to match the Prow and Test Grid color pallette, but otherwise fonts
and text size and spacing and things can match.  Headlamp is built with
Material, so I believe this may be easy to replicate.

## UI Structure

Both Prow and Test Grid seem to link back and forth to each other. In some
basic sense, it seems like Test Grid is "higher level" and can lead a user
down into a particular set of tests a bit more intuitively.  However, Kube
runs tons of jobs in CI, and often there will be links from the GitHub UI
directly to Prow tests.  We will want to keep this in mind and be able
to support the direct linking.

## The Plan: Backend

- Thin proxy if doable
- Caching, if we need it, since it can be quite a lengthy wait for Tests to
  run. It also may be a significant amount of data to query.  If we can
  smartly cache to reasonably short lengths of time so user's aren't waiting
  ages for updates, that may benefit us.
  - No need to prematurely optimize, let's keep this in mind.

# Incremental TODOs

Ordered for incremental progress. Backend proxy first so the frontend has
real data to build against. Frontend is the critical interesting piece, so
we scaffold the backend just enough to unblock UI work, then shift focus
to the frontend and iterate.

## Phase 1: Backend Proxy — Scaffolding

- [ ] 1.1 — Backend directory structure:
  ```
  backend/
    cmd/
      server/
        main.go          # entrypoint
    internal/
      config/
        config.go        # env/flag parsing (listen addr, upstream URLs)
      proxy/
        proxy.go         # reverse proxy / HTTP client logic
      server/
        server.go        # HTTP server setup, middleware, router
        routes.go        # route registration
      middleware/
        cors.go          # CORS for local dev
        logging.go       # request logging
    go.mod
    go.sum
    Makefile
  ```

- [ ] 1.2 — `backend/Makefile`: targets for `build`, `run`, `test`, `lint`
  (use `golangci-lint`), `tidy` (`go mod tidy`).

- [ ] 1.3 — `cmd/server/main.go`: minimal main that loads config, creates
  the server, and starts listening.  Graceful shutdown via signal handling.

- [ ] 1.4 — `internal/config/config.go`: parse configuration from env vars
  and/or flags.  At minimum:
  - `LISTEN_ADDR` (default `:8080`)
  - `PROW_UPSTREAM` (default `https://prow.k8s.io`)
  - `TESTGRID_UPSTREAM` (default `https://testgrid.k8s.io`)
  - `CORS_ALLOWED_ORIGINS` (default `http://localhost:5173` for Vite dev)

- [ ] 1.5 — `internal/server/server.go` + `routes.go`: set up
  `net/http.ServeMux` (or a thin router), register routes. No frameworks
  unless warranted later.

- [ ] 1.6 — `internal/middleware/cors.go`: CORS middleware allowing the
  Vite dev origin.  Simple wrapper, no external dep needed.

- [ ] 1.7 — `internal/middleware/logging.go`: basic structured request
  logging (method, path, status, duration).  Use `log/slog` from stdlib.
